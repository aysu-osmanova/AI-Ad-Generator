# 📝 Project Build Instructions: Multi-Platform AI Ad Generator

This document outlines the step-by-step implementation guide to build, test, and deploy the AI Ad Generator. Follow these stages in order to successfully connect the AI engine, backend server, and frontend interface.

---

## Stage 1: Project Initialization & Prerequisites

Before writing any code, set up your development environment and secure your AI access.

### Create the Project Directory

- Create a main folder named `ai-ad-generator`.
- Inside, create a `server` folder (for backend) and a `public` folder (for frontend).

### Acquire API Keys

To connect to the Gemini API, you will need a simple API key.

1. Go to **Google AI Studio**: https://makersuite.google.com/app/apikey
2. Click **"Create API key"** and copy the generated key.
3. In the `ai-ad-generator` folder, create a new file named `.env`.
4. Add your API key to the `.env` file like this:
   ```bash
   GEMINI_API_KEY=your_api_key_here
   ```
> ⚠️ **Never commit the `.env` file to GitHub.** It is already in your `.gitignore`.

### Initialize the Repository

- Run `git init` in your project folder.
- Create a `.gitignore` file and add the following entries:
  ```
  .env
  node_modules/      # Node.js
  __pycache__/       # Python
  ```

---

## Stage 2: Backend Development (The AI Engine)

Reference your `backend.md` and `AI.md` files to build the server logic.

### Set Up the Server

- In the `ai-ad-generator` directory, run `npm init -y` and `npm install express cors dotenv`.
- Configure the server to serve static files from your `public` folder.

### Build the API Route (`POST /api/generate`)

- Create the endpoint to receive `business_name`, `product_service`, and `mode` from the frontend.

### Implement the Routing Logic

- Add a conditional `if/else` statement to check if the user selected **Express Mode** or **Agency Mode**.

### Integrate the AI Prompts

- Open your `AI.md` file and copy the specific system prompts:
  - **AIDA** framework → Instagram
  - **PAS** framework → Facebook
  - **UGC hook** → TikTok
- **Express Mode:** Combine them into one master prompt returning a single JSON object.
- **Agency Mode:** Set up asynchronous parallel calls using the isolated prompts:
  - Node.js: `Promise.all([...])`
  - Python: `asyncio.gather([...])`

### Format the Response

- Ensure the final output sent back to the client perfectly matches the JSON structure defined in `backend.md`.

---

## Stage 3: Frontend Assembly (The UI)

Reference your `frontend.md` file to build the user interface.

### Create the Static Files

- Inside the `public` folder, create `index.html`, `styles.css`, and `app.js`.

### Build the Layout (`index.html`)

- Create the input form with fields for **Business Name**, **Product**, and **Mode toggle**.
- Create three empty result cards (Instagram, Facebook, TikTok) and set them to **hidden by default**.

### Style the Dashboard (`styles.css`)

- Apply a clean, modern aesthetic.
- Add subtle shadows to the cards.
- Ensure cards stack vertically on mobile screens.

### Connect the Logic (`app.js`)

- Add an event listener to the **Generate** button.
- Write the `fetch()` call to send the form data to your backend `/api/generate` route.
- Parse the returned JSON and inject the text into the respective HTML cards.
- Add loading states (e.g., changing the button text to `"Generating... ⏳"` while waiting for the AI).

---

## Stage 4: Local Testing & Debugging

Verify the system works end-to-end before deploying.

### Run the Server Locally

Start your server and open `http://localhost:3000` in your browser:

```bash
# Node.js
npm start

# Python
python main.py
```

### Test Express Mode

1. Input a test business (e.g., `"Baku Roasters"`, `"Cold Brew Coffee"`).
2. Select **Express Mode** and submit.
3. Verify it returns results quickly (under 3 seconds) and the UI updates properly.

### Test Agency Mode

1. Use the same inputs but select **Agency Mode**.
2. Verify the outputs are noticeably deeper, adhere strictly to the frameworks (AIDA, PAS), and load without crashing.

### Test Edge Cases

- Submit empty inputs and ensure the frontend displays an alert or error message instead of breaking the server.

---

## Stage 5: Deployment & Delivery

Publish your project to fulfill the assignment requirements.

### Push to GitHub

- Commit all your code and push it to a **public GitHub repository**.
- Ensure your `README.md` (containing your project documentation and the 3 required examples) is at the root of the repo.

### Deploy to Railway

1. Log into [Railway.app](https://railway.app).
2. Connect your GitHub account and select your repository to deploy.
3. Go to the Railway dashboard **Variables** settings and securely add your `GEMINI_API_KEY` with the value you got from Google AI Studio.

### Final Verification

- Visit the **live URL** generated by Railway.
- Run one final test generation to ensure the production environment successfully connects to the LLM API.
